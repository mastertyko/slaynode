Local Service Control Room for macOS

A Tahoe-native desktop app that discovers local runtimes, Docker containers, and Homebrew Services, explains what they are, and lets you control the right thing from one surface.

Download for macOS View on GitHub

Features

🎯

Unified Discovery

Automatically finds local development runtimes, Docker containers, Homebrew Services, and workspace-backed service stacks.

⚡

Context-Aware Actions

Use Stop, Force Stop, Restart, Logs, and Open Workspace without leaving the app.

🔍

Smart Recognition

Explains sources, ports, health, workspace paths, and runtime roles instead of showing raw process noise.

📊

Native macOS 26 UX

NavigationSplitView, toolbar search, inspector, restored windows, and menu bar triage all feel first-class on modern macOS.

🔒

Private & Secure

Everything runs locally. No service data leaves your machine, and sensitive command flags are redacted in the UI.

🔄

Release Ready

Ships through GitHub Actions with build-numbered DMG/ZIP artifacts, changelog-backed release notes, and a generated icon system aligned with the current SlayNode identity.

Installation

1

Download the Latest Release

Get the latest signed build from the GitHub Releases page.

https://github.com/mastertyko/slaynode/releases/latest
2

Install the App

Open the DMG and drag SlayNode to your Applications folder.

3

Launch SlayNode

Open the app from Applications or Spotlight. The main dashboard window becomes your local service control center.

open /Applications/SlayNode.app

Requirements

macOS 26.0+
Apple Silicon or Intel
Local services to inspect, such as a runtime, Docker container, or Homebrew Service
