# Security Policy

## Supported Versions

Security fixes target the latest public GitHub release and the `main` branch.

## Reporting A Vulnerability

Use GitHub's private vulnerability reporting for this repository when possible.
If that is unavailable, open a minimal public issue that avoids exploit details
and asks for a private coordination channel.

## Public Release Signing

GitHub release assets are intentionally ad-hoc signed and not notarized. This
repository does not require Apple Developer ID certificates, Apple ID credentials,
or notarization secrets for public releases.
